# 预检查接口与阈值配置实施计划

## Summary

- 目标：在现有 `python-face-demo` 中新增“失败图片预检查接口”，用于在执行重建底库前先扫描整个底库目录并返回可入库/失败文件信息；同时把检测阈值和识别阈值从硬编码改为可通过 `YAML` 文件持久化配置，并提供接口读取与设置。
- 成功标准：
  - 提供独立的预检查接口，扫描 `data/gallery/` 全量图片而不重建索引。
  - 返回预检查成功数、失败数、失败文件列表和失败原因。
  - 新增配置文件，至少支持 `检测阈值` 与 `识别阈值`。
  - 提供接口读取当前配置，并支持接口修改后写回 `YAML` 文件。
  - 接口修改阈值后立即生效，不需要重启 Python 服务。
  - `recognize` 与底层检测逻辑都改为读取统一配置，不再硬编码阈值。

## Assumptions & Decisions

- 决策：预检查接口先只支持“扫描整个底库目录”
  - 原因：与当前 `gallery` 目录驱动的 Demo 形态最一致，实施成本最低
- 决策：阈值配置先管理 `检测阈值` 与 `识别阈值`
  - 原因：这两个参数直接影响当前痛点和识别结果
- 决策：配置文件采用 `YAML`
  - 原因：可读性好，适合直接落在 Python 工程目录中手工查看和修改
- 决策：阈值接口更新后立即生效，不要求重启 Python
  - 原因：更符合调参和预检查的使用场景，减少人工重启成本
- 决策：预检查和重建共用同一套底库扫描与图片分析逻辑
  - 原因：避免“预检查能过、重建又失败”的不一致问题
- 假设：允许在 `requirements.txt` 中新增 `PyYAML` 依赖以支持 YAML 配置读写
- 假设：当前 Demo 仍以单机文件配置为主，不引入数据库或远程配置中心
