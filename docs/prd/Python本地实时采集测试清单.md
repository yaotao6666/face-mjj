# Python 本地实时采集测试清单

## 1. 目标

- 验证不依赖浏览器时，是否可以通过本地 Python 脚本读取摄像头并联通现有活体识别服务。

## 2. 测试前提

- 已启动 `python-face-demo` 服务
- 已完成 `POST /api/gallery/precheck`
- 已完成 `POST /api/gallery/rebuild`
- 本机存在可用摄像头
- 已安装脚本依赖

## 3. 启动命令示例

```bash
python scripts/live_camera_demo.py --server http://127.0.0.1:8002 --camera 0 --interval-ms 500
```

## 4. 测试项

### 4.1 摄像头打开成功

- 步骤：
  - 运行脚本
- 预期：
  - 脚本正常启动
  - 摄像头可持续取帧
  - 若 GUI 可用，则显示预览窗口

### 4.2 摄像头编号错误

- 步骤：
  - 使用不存在的 `--camera` 编号运行脚本
- 预期：
  - 脚本直接提示摄像头打开失败
  - 进程退出

### 4.3 服务未启动

- 步骤：
  - 停止后端服务
  - 启动脚本
- 预期：
  - 摄像头仍可打开
  - 脚本持续提示接口请求失败
  - 预览流程不中断

### 4.4 局域网地址不可达

- 步骤：
  - 传入错误的 `--server` 地址
- 预期：
  - 返回清晰的网络错误信息
  - 不导致脚本崩溃

### 4.5 真人活体通过

- 步骤：
  - 真人正脸站到镜头前
- 预期：
  - `livenessResult` 大概率为 `PASS`
  - 返回活体分值

### 4.6 照片翻拍活体拒绝

- 步骤：
  - 使用打印照片对准摄像头
- 预期：
  - `livenessResult` 应尽量趋向 `REJECT`
  - 可用于观察当前阈值效果

### 4.7 屏幕翻拍活体拒绝

- 步骤：
  - 使用手机或平板展示人脸照片后对准摄像头
- 预期：
  - `livenessResult` 应尽量趋向 `REJECT`

### 4.8 底库命中

- 步骤：
  - 已注册员工站到镜头前
- 预期：
  - `matched=true`
  - 返回 `employeeNo`、`employeeName`、`similarity`

### 4.9 底库未命中

- 步骤：
  - 未注册人员站到镜头前
- 预期：
  - `matched=false`
  - 返回相似度但不返回员工信息

### 4.10 调整抽帧间隔

- 步骤：
  - 分别使用 `--interval-ms 300`、`500`、`1000` 运行脚本
- 预期：
  - 请求节奏随参数变化
  - 可观察到响应频率差异

### 4.11 阈值覆盖测试

- 步骤：
  - 使用 `--threshold` 传入不同识别阈值
- 预期：
  - 命中与未命中结果会随阈值变化

### 4.12 退出释放资源

- 步骤：
  - 按 `q` 退出脚本
  - 再次运行脚本
- 预期：
  - 摄像头资源正常释放
  - 可再次打开

## 5. 记录建议

- 建议记录以下字段：
  - 测试时间
  - 测试人
  - 服务地址
  - 摄像头编号
  - 抽帧间隔
  - 活体结果
  - 识别结果
  - 是否符合预期
  - 异常说明
